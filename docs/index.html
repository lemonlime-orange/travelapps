외국인 여행객이 한국에서 어떤 앱을 설치해야 할지 헤매는 문제에서 출발한 프로젝트라고 소개합니다.
01 · WHY

여행객의 불편은
“앱이 없는 것”이 아니었다

뭘 하더라도 앱을 받아야 하는데,
너무 많아서 무엇을 써야 할지 어렵다.
앱이 너무 많음지도·택시·배달·통역 등
목적마다 다른 앱
낯선 사용 환경현지 서비스와 언어에
익숙하지 않은 관광객
검색할 시간 부족돌발 상황에서는
검색할 시간도 부족
흩어진 정보정보가 여러 사이트와
스토어에 흩어져 있음
문제 정의 · 정보 부족보다 선택 피로가 핵심
외국인 관광객 설문 관련 기사를 본 경험을 출발점으로 설명하고, 핵심 문제를 ‘정보의 양’이 아니라 ‘선택의 어려움’으로 정의했다고 말합니다.

11 · CODE BASICS

복잡한 기능도 조건문·반복문·함수의 조합

⑴

조건문 · if

로그인 여부, 관리자 비밀번호, 다운로드 여부, 리뷰 수, 필수 입력값을 판단합니다.

↻

반복문 · for

주제 버튼, 앱 카드, 주요 기능, 이미지, 리뷰, 안내 단계를 하나씩 화면에 표시합니다.

ƒ

함수 · def

데이터 불러오기, 카테고리 필터, 카드 그리기, 별점 계산을 재사용 가능한 작업으로 나눕니다.

load_apps() # 전체 앱 데이터 불러오기 filter_by_category(df, topic) # 선택한 주제만 남기기 render_app_card(app) # 앱 한 개를 카드로 표시 get_display_rating(id, base) # 기본/실사용 별점 결정
큰 기능을 작은 책임으로 나누어 유지보수 가능하게 구성
비전공자에게는 조건문은 ‘허용 여부 판단’, 반복문은 ‘같은 형식 반복’, 함수는 ‘이름 붙인 작은 작업’으로 설명합니다.

구현·이전

테스트·수정

내 역할

웹앱 코드 작성, 앱 정보 조사·정리·분석, 기능 테스트와 데이터 구조 개선

활용 도구

VS Code · Android Studio · GitHub · Claude · ChatGPT(Codex)

AI 협업의 핵심 · 구체적인 조건 + 실제 실행 확인 + 반복 수정
원고의 주요 프롬프트를 모두 읽기보다는, 요구사항을 구체적으로 전달하고 결과를 직접 테스트한 과정으로 요약해 설명합니다.
13 · ITERATION

오류를 고치며 데이터와 화면의 구조를 배웠다

문제 01 · 메뉴 위치

다른 메뉴를 열면 관리자 메뉴가 중간으로 이동

해결: 공통 화면 흐름과 렌더링 순서를 조정해 항상 같은 위치에서 접근하도록 개선.

배운 점: 화면 요소의 위치도 코드 실행 순서의 결과다.

문제 02 · 데이터 구조

여러 안내 내용을 한 셀에 저장해 로딩이 느려짐

해결: Before You Land 내용을 별도 테이블의 여러 행으로 분리.

배운 점: 데이터는 한 덩어리보다 역할별 행·테이블로 나누는 것이 관리에 유리하다.

문제 03 · 구분자

추천 앱이 2개 이상일 때 ID를 제대로 읽지 못함

해결: 쉼표와 파이프를 모두 인식하고, 숫자가 아닌 값과 중복 ID를 건너뜀.

공통 교훈

“기능이 된다”에서 “데이터가 오래 유지된다”로

입력 검증, 사용자별 데이터, 실패 시 처리, 관리자 편집까지 고려하며 완성도를 높였습니다.

문제 → 원인 추적 → 구조 수정 → 재검증
원고에 있던 두 가지 오류 해결 경험에, 저장소 변경 기록에서 확인한 Situation Helper 구분자 문제를 추가했습니다.
